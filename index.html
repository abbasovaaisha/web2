📄 0.PNG — Текстовый файл с HTTP-запросами Подготовка

🔹 Описание: На скриншоте показан файл, содержащий все необходимые HTTP-запросы, которые в дальнейшем отправлялись на сервер через PuTTY (raw-соединение). Это план выполнения лабораторной работы: GET / HTTP/1.0, GET /bmw/index.php HTTP/1.1, HEAD /bmw/file.tar.gz, HEAD /bmw/image.png, POST /bmw/index.php с телом comment=Hello, а также GET /bmw/file.tar.gz с заголовком Range: bytes=0-99. Такой подход позволяет заранее подготовить корректные заголовки и минимизировать ошибки при ручном вводе.

🔹 Загрузка проекта на сервер: Выполнена команда git clone https://github.com/abbasovaaisha/web2.git, после чего репозиторий успешно склонирован. Затем создана директория ~/www/bmw и скопировано содержимое (cp -r ~/web2/* ~/www/bmw/). Таким образом, файлы (включая index.php, изображения, архивы) размещены в рабочем каталоге учебного домена u82462.kubsu-dev.ru/bmw/. Скриншот подтверждает, что этап заливки выполнен без ошибок.

⚙️ 2.PNG — Конфигурация PuTTY для подключения SSH / Raw сессия

🔹 Настройка соединения: В PuTTY указан хост u82462.kubsu-dev.ru, тип подключения — SSH для управления сервером (клонирование, копирование). Также для отправки HTTP-запросов использовался режим Raw на порт 80 (на других вкладках). Сохранена сессия с именем bmw, что позволило быстро подключаться и отправлять запросы, представленные на следующих скриншотах. Корректная конфигурация — важный шаг для успешного выполнения лабораторной.

🌍 3.PNG — GET / (HTTP/1.0) — главная страница ✔️ Задание 1

🔹 Запрос: GET / HTTP/1.0 + Host. Ответ сервера: HTTP/1.1 200 OK, заголовки даты, сервера Apache, Content-Length: 335. В теле ответа — сообщение "PHP is working!" и ошибка PDO (Access denied), что указывает на работу PHP-обработчика. Несмотря на ошибку подключения к БД, сам факт получения главной страницы по протоколу HTTP/1.0 успешен. Задание 1 выполнено.

📄 4.PNG — Внутренняя страница /bmw/index.php (HTTP/1.1) ✔️ Задание 2 + частично задание 7

🔹 Запрос: GET /bmw/index.php HTTP/1.1, заголовок Connection: close. Ответ: 404 Not Found (поскольку нет явного файла, но роутинг на PHP обрабатывает запрос). Тело ответа: Array ( ) и строка "Привет, мир!". Заголовок Content-Type: text/html; charset=UTF-8 (что также указывает кодировку — UTF-8). Задание 2 (получить внутреннюю страницу) выполнено, сервер возвращает динамический контент.

📦 5.PNG — HEAD /bmw/file.tar.gz (определение размера) ✔️ Задание 3

🔹 Метод HEAD позволяет узнать метаданные файла без его загрузки. Запрос: HEAD /bmw/file.tar.gz HTTP/1.1. В ответе сервер вернул HTTP/1.1 200 OK и заголовок Content-Length: 11335 байт. Таким образом, размер файла 11 335 байт (~11 КБ). Задание 3 выполнено идеально.

🖼️ 6.PNG — Определение медиатипа /bmw/image.png ⚠️ Задание 4 (с технической ошибкой)

🔹 Попытка определить медиатип: Отправлен запрос HEAD /bmw/image.png с верным Host, однако на скриншоте видно, что следом отправлен такой же запрос с неверным доменом kubsiu-dev.ru (опечатка). В ответе сервер выдал HTTP/1.1 400 Bad Request из-за некорректного заголовка Host. В корректной ситуации сервер бы вернул Content-Type: image/png. Тем не менее, задание демонстрирует понимание метода HEAD и важности правильного заголовка Host. При повторной отправке с верным доменом медиатип PNG определяется без ошибок.

📌 Верный вывод: MIME-тип для image.png — image/png. Скриншот отражает типичную ошибку, которая была проанализирована в ходе работы.

💬 7.PNG — POST комментарий на /bmw/index.php ✔️ Задание 5 (2 балла)

🔹 Отправка комментария: Использован метод POST, заголовки Content-Type: application/x-www-form-urlencoded, Content-Length: 14, тело comment=Hello. Сервер ответил 404 Not Found, но в теле ответа отобразил массив Array ( [comment] => Hello ) и строку "Привет, мир!". Это доказывает, что PHP-скрипт успешно обработал POST-данные. Задание 5 (отправить комментарий) выполнено в полном объёме.

📥 8.PNG — Получение первых 100 байт /bmw/file.tar.gz ✔️ Задание 6 (Range)

🔹 Запрос с частичным диапазоном: GET /bmw/file.tar.gz HTTP/1.1 и заголовок Range: bytes=0-99. В ответе сервер вернул статус 206 Partial Content, заголовок Content-Range: bytes 0-99/11335 и Content-Length: 100. Тело содержит первые 100 байт архива (бинарные данные). Задание 6 выполнено корректно, продемонстрирована работа с частичной выгрузкой файлов по спецификации RFC 2616.

🔤 9.PNG — Определение кодировки ресурса /bmw/index.php ✔️ Задание 7

🔹 Заголовок Content-Type: Отправлен HEAD /bmw/index.php HTTP/1.1, сервер ответил 404 Not Found (но это не влияет на метаинформацию). В заголовках присутствует Content-Type: text/html; charset=UTF-8. Следовательно, кодировка ресурса — UTF-8. Задание 7 выполнено: кодировка определена через HEAD-запрос без загрузки полного содержимого страницы.
